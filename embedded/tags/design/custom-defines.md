# Tag `custom.h` Build Defines

Each tag target has a local `inc/custom.h`. That file is included through
`common/core/inc/app.h` and is the tag-level place for published firmware
identity strings, tag constants, bus choices, and a few board pin aliases.

Build membership is controlled by `TAG_MODULES` in `project.mk`. Modules define
their corresponding feature switches through `UDEFS`. For example, selecting
`sensor_accel_adxl362` defines `TAG_SENSOR_ACCEL_ADXL362`.

Most of these switches are tested with `#ifdef` or `#if defined(...)`. For those
switches, the value is not inspected; defining a feature as `FALSE` still counts
as enabled to code that uses `#ifdef FEATURE`.

## Define Reference

| Define | Meaning | Code impacted |
| --- | --- | --- |
| `FIRMWARE_STRING` | Firmware identity reported to host tools. | Copied into `Ack.info.firmware` by `common/core/src/monitor.c` and `BitTag/src/monitor.c`; older local monitors do the same. |
| `BOARD_NAME` | Overrides the generated board name string reported to host tools. | Used in the monitor info string table in `common/core/src/monitor.c` and `BitTag/src/monitor.c`. It normally overrides the `BOARD_NAME` generated by the board template. |
| `QTMONITOR_VERSION` | Minimum host monitor version reported by the tag. | `common/core/src/monitor.c` sets `Ack.info.qtmonitor_min_version`; defaults to `1.5` if absent. |
| `PROTOBUFSIZE` | Size of the shared protobuf request/ack buffer. | `common/core/src/handlers.c` allocates `ProtoBuf`, publishes `protobuf_size`, and statically checks `Ack_size` and `Req_size` against this value. Defaults to `2056` if absent. |
| `TAG_RTC_RV3028` | Selects the Micro Crystal RV-3028 RTC implementation. | `common/rtc/inc/rtc_api.h` includes `rv3028.h` from `common/rtc/inc`; `common/rtc/src/rtc_device.c` provides the weak default tag RTC device descriptor; `common/rtc/src/rtc_rv3028.c` compiles descriptor-driven RV-3028 register access and initialization. |
| `EXT_FLASH_SIZE` | External flash capacity in bytes for older storage bindings. | Active AT25XE/MX25R bindings now take geometry from the storage chip headers and tag/family `TagStorageDevice` descriptors. IMUTagBreakout's MX25L path still uses this define until that storage binding is cleaned up. |
| `TAG_HAS_EXTERNAL_FLASH` | Marks targets with external flash modules. | Generated by flash modules for compatibility and tag-level selection. Common core reset and monitor reporting now call weak external-flash hooks, so core code does not branch on this define. |
| `TAG_FLASH_AT25XE`, `TAG_FLASH_MX25L`, `TAG_FLASH_MX25U12843`, `TAG_FLASH_MX25R` | Selects the external flash chip driver compiled by the module. | Driver modules in `common/storage/src`; use these for chip-specific code rather than testing the generic flash capability. |
| `TAG_STORAGE_SPI_DMA_BLOCK_READ` | Enables the DMA implementation of `tagStorageSpiBlockRead()` for long external-flash data phases. | `common/storage/inc/storage_spi.h`. Requires a valid storage SPI DMA descriptor and `STM32_DMA_REQUIRED` in the target or family `mcuconf.h` so ChibiOS links the STM32 DMA support code. Command and address phases still use the conservative SPI helper. |
| `TAG_STORAGE_SPI_DMA_BLOCK_WRITE` | Enables the DMA implementation of `tagStorageSpiBlockWrite()` for long external-flash data phases. | `common/storage/inc/storage_spi.h`. Has the same build requirements as `TAG_STORAGE_SPI_DMA_BLOCK_READ`; the write path also runs RX DMA to drain the receive side and avoid SPI overrun. |
| `TAG_STORAGE_SPI_MEASURE_LINE` | Optional output line pulsed around storage SPI block-transfer phases. | `common/storage/inc/storage_spi.h`. Used for logic-analyzer timing during storage bring-up; leave undefined for normal builds. |
| `TAG_SENSOR_ACCEL_ADXL362` | Selects ADXL362 accelerometer support. | Generated by `sensor_accel_adxl362`; the ADXL362 driver and self-test live in `common/sensors/accel`, while active tag/family `devices.c` files own descriptor binding and reset/deinit hooks. `BitTag-legacy` keeps the frozen local ADXL362 path. |
| `TAG_SENSOR_ACCEL_LIS2DU12` | Selects LIS2DU12 accelerometer support and its self-test hook. | Generated by the CompassTag family manifest. `families/CompassTag/src/lis2du12.c` provides the driver and `families/CompassTag/src/lis2du12_test.c` provides the hook called by the shared test driver. |
| `USE_LIS2DU12` | Selects LIS2DU12 accelerometer support in older/tag-local paths. | Retired from active CompassTag custom headers. The CompassTag family manifest now selects the LIS2DU12 module and `families/CompassTag/src/devices.c` owns the USART-backed binding. |
| `LINE_ACCEL_CS` | Pin alias for the accelerometer chip-select line. | Used by descriptor-backed accelerometer bindings, tag/family `devices.c` standby-pull hooks, and local LIS2DU12 drivers. Some tags rely on the board file for this line instead of defining it locally. |
| `LINE_ACCEL_INT` | Pin alias for the accelerometer interrupt/wakeup line. | Used by BitTag and IMUTagBreakout wakeup paths to configure wakeup edge handling. BitPresTag uses the board-generated `LINE_WKUP1` name directly. |
| `LOG_ACK_MEASURE_LINE` | Optional output line pulsed high while a tag builds and encodes a data-log acknowledgement. | Used for logic-analyzer profiling of monitor download latency. IMUTagv1 can define this as `LINE_IMU_TRG_TEST` to use PA4. |
| `ACCEL_WAKEUP_SOURCE` | Selects which STM32 wakeup source is connected to the accelerometer wakeup line; defaults to `4` when absent. | Tag/family `devices.c` files use it to select the matching `PWR_CR4_WP*` edge bit and `PWR_CR3_EWUP*` enable bit. BitPresTag breakout hardware and the active CompassTag family use wakeup source `1`. |
| `TAG_SENSOR_PRESSURE_LPS27` | Selects LPS27 pressure sensor support. | `common/sensors/pressure/src/lps27.c` compiles the parameterized LPS27 driver and `common/sensors/pressure/src/lps27_test.c` provides a weak self-test placeholder. Active tags bind the real LPS27 device and test in `devices.c`. |
| `LPS_SPI`, `LPS_USART`, `LPS_LOW_POWER` | Retired pressure-driver configuration switches. | Active pressure drivers now take `TagPressureDevice` descriptors from tag/family `devices.c`. Older LPS22/LPS33/BMP implementations that used these switches live under `common/sensors/archive/pressure`. |
| `TAG_SENSOR_PRESSURE_LPS22HH` | Selects LPS22HH pressure sensor support and the IMUTagBreakout LPS22HH self-test hook. | Generated by `sensor_pressure_lps22hh`; `common/sensors/pressure/src/lps22hh.c` provides the driver and `IMUTagBreakout/src/lps22hh_test.c` provides the hook called by the shared test driver. |
| `USE_LPS22HH` | Selects the LPS22HH pressure path in `IMUTagBreakout`. | Used by IMUTagBreakout-local sensor code. Prefer module-generated `TAG_SENSOR_PRESSURE_LPS22HH` in shared code. |
| `USE_STOP1` | Selects Stop1 instead of Stop2 for the shared `stopMilliseconds()` low-power delay helper. | Used by targets that need peripherals unavailable in Stop2, such as IMUTagBreakout's LPTIM2-triggered IMU path. |
| `CONFIG_HAS_HIBERNATE` | Indicates whether the tag's stored configuration includes hibernation intervals. Defaults to enabled when absent. | Common state-machine and persistent sizing code use this to avoid referencing `sconfig.hibernate` for tags such as IMUTagBreakout that do not store hibernation intervals. |
| `TAG_STATUS_FIXED_VDD100` | Overrides the reported and state-marker supply voltage with a fixed value in hundredths of a volt, while still using the ADC for die temperature when status is sampled. | `common/core/src/monitor.c` applies this to monitor status responses and skips ADC sampling while acquisition is active; `common/core/src/persistent.c` applies it to state markers after taking the ADC sample needed for temperature. |
| `TAG_SENSOR_MAG_AK09940A` | Selects AK09940A magnetometer support. | The common AK09940A module supplies the register driver, default shim, and weak self-test hook. Active CompassTag variants bind the concrete `TagRegisterDevice` in family `devices.c`; IMUTagBreakout binds its mixed SPI wiring in `IMUTagBreakout/src/devices.c`. |
| `LINE_MAG_CS`, `LINE_MAG_SCK`, `LINE_MAG_MISO`, `LINE_MAG_MOSI`, `LINE_MAG_TRG` | Standard magnetometer bus and trigger line names for AK09940A board wiring. | `common/sensors/mag/src/ak09940a_shim.c` uses the full bus-name set only for the weak default legacy binding. Tags with shared or mixed wiring can provide a strong `tagAk09940aDevice()` descriptor from `devices.c` instead of defining board aliases. |
| `TAG_SENSOR_IMU_LSM6DSV16X` | Selects LSM6DSV16X IMU support. | The common IMU module supplies the descriptor-driven register driver and weak accelerometer self-test hook. The IMUTag family binds the concrete `TagLsm6dsv16xDevice` in `families/IMUTag/src/devices.c`, accepts both `RUN_LSM6DSV16X` and the legacy `RUN_AIS2` monitor request, and reports `LSM6DSV16X_FAILED` on failure. |
| `USE_BMP` | Selects BMP pressure sensor support in an older prototype. | Archived pressure-driver/prototype code only. Not used by active targets. |
| `ACCEL_USART` | Selects the USART-backed LIS2DU12 transfer variant. | Compass family LIS2DU12/power code and older IMU power-control files. |
| `SENSOR_CALIBRATION` | Enables the runtime sensor calibration command/state. | `common/core/src/state_machine.c` exposes and enters `TagState_CALIBRATE`; `common/core/src/monitor.c` accepts `Req_calibrate` and `Req_caldata`; `common/core/inc/app.h` exposes calibration declarations. Active CompassTag variants select this from the family manifest rather than `custom.h`. |
| `CALIBRATION_CONSTANTS` | Enables read/write access for stored calibration constants. | `common/core/src/monitor.c` accepts `Req_write_calibration` and `Req_read_calibration`. Active CompassTag variants select this from the family manifest rather than `custom.h`. |
| `SENSOR_CONSTANTS` | Reports sensor scale constants in monitor info. | `common/core/src/monitor.c` copies `ACCEL_CONSTANT` and `MAG_CONSTANT` to `Ack.info`. Active CompassTag variants select this from the family manifest rather than `custom.h`. |
| `ACCEL_CONSTANT` | Accelerometer scale constant reported to host tools. | Only read when `SENSOR_CONSTANTS` is defined, in `common/core/src/monitor.c`. |
| `MAG_CONSTANT` | Magnetometer scale constant reported to host tools. | Only read when `SENSOR_CONSTANTS` is defined, in `common/core/src/monitor.c`. |
| `COMPASS_TAG` | Enables compass-tag-specific sample handling in the AT25 compass variant. | `CompassTagAT25/src/sensors.c`. This is not currently set in every compass target, so treat it as local to that implementation rather than a general compass capability flag. |
| `SWAP_I2C` | Selects swapped RTC I2C line behavior for breakout/base-board layouts with RTC_SDA and RTC_SCL routed to the opposite pins. | `common/core/src/pwr.c` uses it when building the software I2C fallback `I2CConfig`. |
| `TAG_DEBUG_LOG` | Enables in-firmware debug message buffering and retrieval. | Generated by the `debug_log` module. `common/core/src/debug_log.c` owns the memory stream buffer; `common/core/src/handlers.c` initializes it; `common/core/src/monitor.c` reports availability and serves `Req_debug`; drivers can emit optional messages through `debug_log_printf()`. |
| `DEBUG_MESSAGES` | Legacy compatibility alias for debug logging. | Generated by the `debug_log` module for older source that still checks this name. Do not add it to new `custom.h` files; select the `debug_log` module instead. |
| `STM32_MSIRANGE_FAST` | Fast MSI clock setting used during monitor/protobuf operations. | Used with `RANGE_MULTIPLIER` by `common/core/src/handlers.c` and `BitPresTag/src/datalog.c` to temporarily speed up the MCU. |
| `RANGE_MULTIPLIER` | Timer prescaler multiplier for the fast MSI mode. | Enables the optional fast/slow MSI switch blocks in `common/core/src/handlers.c`; `BitPresTag/src/datalog.c` also uses it directly. |
| `FLASH_WS_SLOW` | Flash wait-state setting for the normal clock mode. | Used with `RANGE_MULTIPLIER` in `common/core/src/handlers.c` and `BitPresTag/src/datalog.c`. |
| `FLASH_WS_FAST` | Flash wait-state setting for the fast clock mode. | Used with `RANGE_MULTIPLIER` in `common/core/src/handlers.c` and `BitPresTag/src/datalog.c`. |
| `BITTAG_V6` | Identifies BitTag V6/Nucleo-style firmware in `custom.h`. | No current source use was found outside `custom.h`; it appears informational or obsolete. |
| `LINE_STEVAL_CS` | Pin alias for the pressure/sensor board chip-select line. | Used by PresTag and older archived pressure-driver code. BitPresTag uses the board-generated `LINE_LPS_CS` name directly. Prefer a tag/family `TagPressureDevice` binding over direct source checks for this alias. |

## Retired Or Prototype-Only Defines

The following defines appear in `custom.h` files for targets that are not
currently built, or in older driver code that is not selected by active module
manifests:

- `USE_BMP`: archived BMP pressure-sensor prototype path.
- `BITTAG_V6`: currently no source use outside `custom.h`.
- `RV3028_RTC`, `EXTERNAL_FLASH`, `USE_ADXL362`, `USE_LIS2DU12`,
  `ACCEL_USART`, `USE_LPS27`, and
  `USE_AK09940A`: retired compatibility names for module-owned features; use
  the corresponding `TAG_*` switches above.
- `USE_LPS22`, `USE_LPS33`, `USE_AIS2`, `USE_LIS2`, `USE_OPT3002`, and
  `USE_MAG` are archived or prototype-era sensor selectors and are not defined
  by active `custom.h` files.

## Maintenance Notes

- Prefer capability names that match one sensor or one behavior. If a tag needs
  a different bus or pin mapping, document it here when adding the define.
- Prefer presence-only switches without values, or update the consuming code to
  use value checks consistently. Do not write `#define FEATURE FALSE` if the
  code uses `#ifdef FEATURE`.
- Select source files through `TAG_MODULES` when the feature controls build
  membership. Add module-owned feature switches in the module fragment through
  `UDEFS`; use `custom.h` for tag constants, bus choices, board aliases, and
  behavior that is not equivalent to selecting a module.
- When adding a new define, update this document with the defining tags and the
  source files that use it.
