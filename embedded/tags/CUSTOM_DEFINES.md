# Tag `custom.h` Build Defines

Each tag target has a local `inc/custom.h`. That file is included through
`common/core/inc/app.h` and is the tag-level place for build-time feature switches,
published firmware identity strings, and a few board pin aliases.

Most of these switches are tested with `#ifdef` or `#if defined(...)`. For those
switches, the value is not inspected. This matters: `#define LPS_LOW_POWER FALSE`
still counts as enabled to code that uses `#ifdef LPS_LOW_POWER`.

These are the `custom.h` defines used by the active tag targets in

history when consolidating old prototype code, but should not be treated as the
current preferred model.

## Define Reference

| Define | Meaning | Code impacted |
| --- | --- | --- |
| `FIRMWARE_STRING` | Firmware identity reported to host tools. | Copied into `Ack.info.firmware` by `common/core/src/monitor.c` and `BitTag/src/monitor.c`; older local monitors do the same. |
| `BOARD_NAME` | Overrides the generated board name string reported to host tools. | Used in the monitor info string table in `common/core/src/monitor.c` and `BitTag/src/monitor.c`. It normally overrides the `BOARD_NAME` generated by the board template. |
| `QTMONITOR_VERSION` | Minimum host monitor version reported by the tag. | `common/core/src/monitor.c` sets `Ack.info.qtmonitor_min_version`; defaults to `1.5` if absent. |
| `PROTOBUFSIZE` | Size of the shared protobuf request/ack buffer. | `common/core/src/handlers.c` allocates `ProtoBuf`, publishes `protobuf_size`, and statically checks `Ack_size` and `Req_size` against this value. Defaults to `2056` if absent. |
| `RV3028_RTC` | Selects the Micro Crystal RV-3028 RTC implementation. | `common/core/inc/app.h` includes `rv3028.h` from `common/rtc/inc`; `common/rtc/src/rtc_rv3028.c` compiles RV-3028 register access and initialization. |
| `EXT_FLASH_SIZE` | External flash capacity in bytes. | External-flash log bounds in tag-local `datalog.c` files; `common/core/src/monitor.c` reports `Ack.info.extflashsz`; `common/src/at25xe.c`, `common/src/mx25l.c`, and `common/src/mx25r.c` derive sector counts from it. |
| `EXTERNAL_FLASH` | Enables external-flash support paths. | Power setup and low-power handling in `common/src/pwr.c` and `BitTag/src/bt_pwr.c`; erase/recovery paths in `common/core/src/state_machine.c`; self-test in `common/src/test.c`; monitor status/info fields in `common/core/src/monitor.c`; local tag `pwr.c`, `state_machine.c`, `test.c`, and `datalog.c` variants use the same switch. |
| `USE_ADXL362` | Selects ADXL362 accelerometer support. | Includes and reset paths in `common/core/src/main.c` and `common/core/src/state_machine.c`; SPI power path in `common/src/pwr.c` and `BitTag/src/bt_pwr.c`; ADXL362 self-test in `common/src/test.c`; local state machines and tests use the same guard. |
| `USE_ADXL367` | Selects ADXL367 accelerometer support in older BitTagNG-style code. | Includes and reset paths in `common/core/src/main.c` and `common/core/src/state_machine.c`; shared power setup in `common/src/pwr.c`; older `BitTagNG` tests and local state logic. |
| `USE_LIS2DU12` | Selects LIS2DU12 accelerometer support. | Reset/deinit paths in `common/core/src/main.c` and `common/core/src/state_machine.c`; shared power setup in `common/src/pwr.c`; compass/IMU local `lis2du12.c`, `test.c`, `state_machine.c`, and `pwr.c`. |
| `LINE_ACCEL_CS` | Pin alias for the accelerometer chip-select line. | Used by `common/src/ADXL362.c`, `common/src/ADXL367.c`, `common/src/pwr.c`, `BitTag/src/bt_pwr.c`, and local LIS2DU12 drivers. Some tags rely on the board file for this line instead of defining it locally. |
| `LINE_ACCEL_INT` | Pin alias for the accelerometer interrupt/wakeup line. | Used by `common/src/pwr.c`, `BitTag/src/bt_pwr.c`, `BitTag/src/bt_state_run.c`, and local `state_run.c`/`pwr.c` files to configure wakeup edge handling. |
| `USE_LPS27` | Selects LPS27 pressure sensor support. | `common/src/lps27.c` compiles LPS27 access paths; `common/src/test.c` includes the LPS self-test when this or related LPS switches are present. |
| `LPS_SPI` | Selects SPI access for supported LPS pressure sensors. | `common/src/lps27.c`, `common/src/lps22.c`, and older `common/src/lps33.c`; `common/src/pwr.c` and `BitTag/src/bt_pwr.c` add the LPS SPI power/chip-select path. |
| `LPS_USART` | Selects USART-style bit access for LPS27. | `common/src/lps27.c` compiles the USART send/receive helpers used by `BitPresTag`-style builds. |
| `LPS_LOW_POWER` | Enables low-power behavior in older LPS22/LPS33 drivers. | `common/src/lps22.c` and `common/src/lps33.c`. Current active LPS27 code does not use this switch. Because the code uses `#ifdef`, defining it as `FALSE` still enables the guarded block where it is used. |
| `USE_LPS22HH` | Selects the LPS22HH self-test path in `IMUTagBreakout`. | Used by `IMUTagBreakout/src/test.c`. The common LPS22 driver uses `USE_LPS22`, not `USE_LPS22HH`, so this is currently a tag-local convention. |
| `USE_AK09940A` | Selects AK09940A magnetometer-related power behavior. | Compass local `pwr.c` files use it to power/configure the magnetometer bus. The AK09940A source itself is selected by the module list rather than by this define. |
| `USE_BMP` | Selects BMP pressure sensor support in an older prototype. | `PresTagBMPBreakout/src/test.c`; older BMP driver code uses `LINE_STEVAL_CS` for chip select. Not used by active targets. |
| `ACCEL_USART` | Selects the USART-backed LIS2DU12 transfer variant. | Compass/IMU local `lis2du12.c` and `pwr.c` files. |
| `SENSOR_CALIBRATION` | Enables the runtime sensor calibration command/state. | `common/core/src/state_machine.c` exposes and enters `TagState_CALIBRATE`; `common/core/src/monitor.c` accepts `Req_calibrate` and `Req_caldata`; `common/core/inc/app.h` exposes calibration declarations. |
| `CALIBRATION_CONSTANTS` | Enables read/write access for stored calibration constants. | `common/core/src/monitor.c` accepts `Req_write_calibration` and `Req_read_calibration`. Some files define this as `TRUE;`; that is harmless for current `#ifdef` usage but should not be copied into code that evaluates the macro value. |
| `SENSOR_CONSTANTS` | Reports sensor scale constants in monitor info. | `common/core/src/monitor.c` copies `ACCEL_CONSTANT` and `MAG_CONSTANT` to `Ack.info`. |
| `ACCEL_CONSTANT` | Accelerometer scale constant reported to host tools. | Only read when `SENSOR_CONSTANTS` is defined, in `common/core/src/monitor.c`. |
| `MAG_CONSTANT` | Magnetometer scale constant reported to host tools. | Only read when `SENSOR_CONSTANTS` is defined, in `common/core/src/monitor.c`. |
| `COMPASS_TAG` | Enables compass-tag-specific sample handling in the AT25 compass variant. | `CompassTagAT25/src/sensors.c`. This is not currently set in every compass target, so treat it as local to that implementation rather than a general compass capability flag. |
| `SWAP_I2C` | Selects swapped I2C line behavior for some breakout layouts. | Local `pwr.c` variants use it for I2C line assignment. Some local files also define it internally when absent, so review the local `pwr.c` before assuming `custom.h` is the only control point. |
| `DEBUG_MESSAGES` | Enables in-firmware debug message buffering and retrieval. | `common/core/inc/app.h` includes memory stream support; `common/core/src/handlers.c` allocates the debug buffer and logs protobuf activity; `common/core/src/monitor.c` reports/debugs messages; some drivers emit extra debug output. |
| `STM32_MSIRANGE_FAST` | Fast MSI clock setting used during monitor/protobuf operations. | Used with `RANGE_MULTIPLIER` by `common/core/src/handlers.c` and `BitPresTag/src/datalog.c` to temporarily speed up the MCU. |
| `RANGE_MULTIPLIER` | Timer prescaler multiplier for the fast MSI mode. | Enables the optional fast/slow MSI switch blocks in `common/core/src/handlers.c`; `BitPresTag/src/datalog.c` also uses it directly. |
| `FLASH_WS_SLOW` | Flash wait-state setting for the normal clock mode. | Used with `RANGE_MULTIPLIER` in `common/core/src/handlers.c` and `BitPresTag/src/datalog.c`. |
| `FLASH_WS_FAST` | Flash wait-state setting for the fast clock mode. | Used with `RANGE_MULTIPLIER` in `common/core/src/handlers.c` and `BitPresTag/src/datalog.c`. |
| `BITTAG_V6` | Identifies BitTag V6/Nucleo-style firmware in `custom.h`. | No current source use was found outside `custom.h`; it appears informational or obsolete. |
| `LINE_STEVAL_CS` | Pin alias for the pressure/sensor board chip-select line. | Used by LPS/BMP drivers and power code: `common/src/lps27.c`, `common/src/lps22.c`, `common/src/lps33.c`, `common/src/pwr.c`, `BitTag/src/bt_pwr.c`, and local pressure-tag variants. |

## Retired Or Prototype-Only Defines

The following defines appear in `custom.h` files for targets that are not
currently built, or in older driver code that is not selected by active module
manifests:

- `USE_ADXL367`: older BitTagNG accelerometer path.
- `USE_BMP`: BMP pressure-sensor prototype path.
- `BITTAG_V6`: currently no source use outside `custom.h`.
- `USE_LPS22`, `USE_LPS33`, `USE_AIS2`, `USE_LIS2`, `USE_OPT3002`, and
  `USE_MAG` are referenced in common test/driver code but are not defined by
  active `custom.h` files.

## Maintenance Notes

- Prefer capability names that match one sensor or one behavior. If a tag needs
  a different bus or pin mapping, document it here when adding the define.
- Prefer presence-only switches without values, or update the consuming code to
  use value checks consistently. Do not write `#define FEATURE FALSE` if the
  code uses `#ifdef FEATURE`.
- Prefer selecting source files through `TAG_MODULES` when the feature controls
  build membership. Use `custom.h` for behavior that must be compiled inside a
  shared source file.
- When adding a new define, update this document with the defining tags and the
  source files that use it.
